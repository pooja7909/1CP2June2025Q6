① 2D Lists (Two-Dimensional Arrays)
A list of lists — like a table where each row is a record and each column is a field.
📖 What is a 2D list?
A 2D list is a list where each element is itself a list. Think of it like a spreadsheet:
Index Column [0] — Key Column [1] — Signal
[0]"A2B3C5"3.2
[1]"A7B30"4.1
[2]"BAD!!"9.9
Python — creating a 2D list
# A 2D list: outer list contains records (rows)
# Each record is an inner list: [key, signal]
records = [
    ["A2B3C5", 3.2],   # record 0
    ["A7B30",  4.1],   # record 1
    ["BAD!!",  9.9],   # record 2  (invalid!)
]
🎯 Accessing Elements — records[row][column]
Always use two indices: first the row, then the column.
Python — indexing
# records[ROW][COLUMN]
records[0][0]   # → "A2B3C5"    (row 0, col 0 = first key)
records[0][1]   # → 3.2         (row 0, col 1 = first signal)
records[1][0]   # → "A7B30"    (row 1, col 0 = second key)
records[2][1]   # → 9.9         (row 2, col 1 = third signal)

# Column 0 = key, Column 1 = signal — ALWAYS
key    = records[0][0]
signal = records[0][1]
💡
Remember: Python counts from 0, not 1. The first row is index [0], the second is [1], etc.
🔁 Looping Through All Records
The most important pattern — this is what the main loop always looks like:
Python — the standard loop pattern
# ✅ CORRECT — works for ANY number of records
for record in records:
    key    = record[0]   # extract the key
    signal = record[1]   # extract the signal
    print(key, signal)

# ❌ WRONG — hardcoded 20, breaks if records change
for i in range(20):
    print(records[i][0])
⚠️
Exam trap: The question says "work even if number of records changes". Using for record in records satisfies this. Using range(20) does NOT — you will lose marks!
📐 Useful 2D List Operations
Python
# How many records are there?
len(records)        # → 3  (number of rows)

# How many fields in each record?
len(records[0])     # → 2  (key + signal)

# Loop with row number (if you need the index)
for i, record in enumerate(records):
    print(f"Record {i}: {record[0]}")

WRITEWrite a function digit_sum(text) that returns the sum of all digit characters in a string. E.g. digit_sum("A2B7") returns 9.[3]
Loop over each character. If it's a digit (use .isdigit()), convert it to int and add to a running total. Return the total.
Answer
def digit_sum(text):
    total = 0
    for char in text:
        if char.isdigit():
            total += int(char)
    return total

isalnum() → True if ALL characters are letters OR digits.

"A5".isalnum() → True (letter + digit = alphanumeric)

Python — function anatomy (annotated)

⑥ call — use the name with () to run it

Return False immediately when a condition fails — don't nest if/else:

④ The Check Digit Algorithm
This specific algorithm is the heart of Q6. Understand every step.
📋 The Rule — Three Steps
Step 1: Extract all digit characters from the key (ignore letters)
↓
Step 2: Sum all those digits
↓
Step 3: Check digit = right-most digit of the sum (= sum % 10)
💡
Key insight: sum % 10 always gives the right-most (units) digit. 13 % 10 = 3, 27 % 10 = 7, 5 % 10 = 5.
🔍 Worked Examples — All Four from the Question
KeyDigits extractedSumSum % 10Last charValid?
A2B3C52, 3555✅
A7B307, 31000✅
A111111✅
A274B32, 7, 41333✅
X9Y2Z49, 21114❌ 1≠4
B5C85558❌ 5≠8
💻 Python Implementation — get_check_digit()
Python — fully annotated
def get_check_digit(key):
    """Returns the expected check digit for a key (as an integer)."""
    total = 0             # running sum of digits

    for char in key:       # loop over every character
        if char.isdigit():  # only process digits (skip letters)
            total += int(char)  # add digit to total

    return total % 10      # right-most digit = total mod 10

# Testing:
print(get_check_digit("A2B3C5"))  # → 5
print(get_check_digit("A274B3"))  # → 3
print(get_check_digit("A7B30"))   # → 0

TRACETrace get_check_digit("A11") step by step. Show the value of total after each character is processed.[3]
Loop over "A", "1", "1". For each, check isdigit(). Track total.
Answer
char = "A" → isdigit()? No → total stays 0
char = "1" → isdigit()? Yes → total = 0 + 1 = 1
char = "1" → isdigit()? Yes → total = 1 + 1 = 2...
Wait — "A11": the digits are both 1s, sum = 1+1 = 2... but the table says check digit = 1!

Look again at the table: A11 has digits: just the first 1 (the key ends in "1", the check digit IS "1"). The sum of digits in "A11" is 1 (from first 1) + 1 (from second 1)... hmm, that gives 2, not 1.

Actually in "A11": digits are 1, 1. Sum = 2. Check digit = 2. But the table shows 1. Let's re-read: the table says sum is just "1" for A11. This means the check digit in "A11" already IS the last character. The key is 3 chars: A, 1, 1. Sum of digits in the key including the last digit: but the check digit IS the last char. So we sum ALL digits... which gives 1+1=2. This is why the example "A11" is a special case — only the digit before the check digit matters?

Re-reading the question: You sum ALL digits (including the check digit itself), then the last digit of the sum should equal the last character. For "A11": digits are 1 and 1, sum = 2, check = 2. But last char = 1. So A11 should be INVALID by this reading... yet the paper shows it as valid.

The correct interpretation: sum ALL digit characters, then check if sum % 10 == int(last_char). For "A11": 1+1=2, 2%10=2. Last char is "1". 2≠1 → INVALID. The paper's "A11" row appears to sum only the non-check digits. Use the paper's rule: the check digit is the last character, and it is NOT included in the sum.
# Correct implementation:
def get_check_digit(key):
    total = 0
    for char in key[:-1]:   # exclude LAST char (the check digit itself)
        if char.isdigit():
            total += int(char)
    return total % 10
⚠️
Important clarification: Based on the question's examples (A11 is valid), the digits summed are from key[:-1] (all chars EXCEPT the last). The last char IS the check digit being validated. Some implementations sum all digits — both approaches may score marks, but matching the paper's examples is safest.

⑤ Writing Validation Functions
Putting together everything you know into clean, working validation code.
🔑 Valid key — 3 conditions
  • 1Minimum length: len(key) >= 3
  • 2Alphanumeric only: key.isalnum()
  • 3Last char = check digit: int(key[-1]) == get_check_digit(key)
⚠️
Check condition 2 FIRST — if the key has symbols, key[-1] might not be a digit and calling int() on it would crash!
📡 Valid signal — 1 condition
A real number where 1.0 ≤ signal ≤ 5.0
Python
def is_valid_signal(signal):
    return 1.0 <= signal <= 5.0

# Tests:
is_valid_signal(3.5)  # True
is_valid_signal(1.0)  # True (boundary!)
is_valid_signal(5.0)  # True (boundary!)
is_valid_signal(0.9)  # False
is_valid_signal(5.1)  # False
💻 Complete is_valid_key() — annotated
Python — fully annotated
def is_valid_key(key):
    """Returns True if key is valid, False otherwise."""

    # Condition 1: minimum length of 3
    if len(key) < 3:
        return False

    # Condition 2: alphanumeric only
    if not key.isalnum():
        return False

    # Condition 3: last char must be a digit
    last = key[-1]
    if not last.isdigit():
        return False

    # Condition 3 (cont): check digit must match
    if int(last) != get_check_digit(key):
        return False

    return True   # all conditions passed!

# Tests:
is_valid_key("A2B3C5")  # True
is_valid_key("AB")      # False (too short)
is_valid_key("A!3")     # False (has !)
is_valid_key("X9Y2Z4") # False (wrong check)

Sum only the digits in a string — used in check digit and similar algorithms.

# ================================================================
# IGCSE CS Paper 2 — Q6 Solution
# Validates records stored in a 2D array
# Fields: [key (str), signal (float)]
# ================================================================


# ── Function 1: Calculate expected check digit ──────────────────
# Sums all digits in key (excluding last char), returns units digit
def get_check_digit(key):
    total = 0
    for char in key[:-1]:         # exclude last character
        if char.isdigit():
            total += int(char)
    return total % 10


# ── Function 2: Validate a key ──────────────────────────────────
# Returns True if key meets all three conditions, False otherwise
def is_valid_key(key):
    # Condition 1: minimum length of 3
    if len(key) < 3:
        return False

    # Condition 2: letters and digits only (alphanumeric)
    if not key.isalnum():
        return False

    # Condition 3: last character must be a valid check digit
    last_char = key[-1]
    if not last_char.isdigit():
        return False
    if int(last_char) != get_check_digit(key):
        return False

    return True   # all conditions satisfied


# ── Function 3: Validate a signal reading ───────────────────────
# Returns True if signal is a real number between 1.0 and 5.0
def is_valid_signal(signal):
    return 1.0 <= signal <= 5.0


# ── 2D Array of records ──────────────────────────────────────────
# Each record: [key (string), signal (float)]
records = [
    ["A2B3C5", 3.2],
    ["A7B30",  4.1],
    ["A11",    2.7],
    ["A274B3", 1.5],
    ["X9Y2Z4", 1.5],   # invalid key
    ["B5C8",   3.3],   # invalid key
    ["A7B30",  6.0],   # invalid signal
    ["!",      0.5],   # both invalid
]


# ── Main program: check every record ────────────────────────────
print("Invalid records:")
print("-" * 40)

for record in records:           # works for any number of records
    key    = record[0]
    signal = record[1]

    key_valid    = is_valid_key(key)
    signal_valid = is_valid_signal(signal)

    if not key_valid or not signal_valid:
        print(f"Key: {key}  Signal: {signal}")
        if not key_valid:
            print("  Invalid key")
        if not signal_valid:
            print("  Invalid signal")

A 2D list called stock stores inventory records. Each record has two fields: a product code (string) and a quantity (integer).

A valid product code must:
• Be exactly 6 characters long

def is_valid_score(score):
    return 0 <= score <= 100

def is_valid_grade(grade):
    return grade in ["A", "B", "C", "D", "F"]

def is_valid_id(sid):
    if len(sid) != 5:
        return False
    if sid[0] != "S":
        return False
    if not sid[1:].isdigit():
        return False
    return True
